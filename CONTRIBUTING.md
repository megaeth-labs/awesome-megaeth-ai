# Contributing to Awesome MegaETH AI

Thank you for helping grow the MegaETH AI ecosystem!

## Adding a New Item

1. Make sure the item is related to AI and MegaETH development
2. Add your item to the appropriate category in alphabetical order
3. Use the format: `[name](link) - Brief description.`
4. Keep descriptions concise — one to two sentences max
5. Check your spelling and grammar

## Quality Standards

- The project should be actively maintained
- Documentation should be available
- The project should provide clear value to MegaETH developers
- Code should be open source or have a publicly accessible API/endpoint

## What Counts as MegaETH-Related?

- Skills, agents, or tools that specifically target MegaETH (chain ID 4326 / 6343)
- Projects that leverage MegaETH-specific features (`eth_sendRawTransactionSync`, mini-blocks, MegaEVM gas model, volatile data access, etc.)
- Cross-chain tools with explicit MegaETH support
- Educational content focused on building on MegaETH

## What NOT to Submit
- **No token promotions.** This list is for developer tools and resources, not token launches or investment opportunities.
- **No paid promotions.** Submissions should be genuine recommendations, not advertisements.
- **No vaporware.** Projects must have working code and be usable by developers today.
- **No testnet automation scripts.** Faucet bots, airdrop farmers, and similar tools are not developer resources.

## Submitting an AI Coding Skill

If your submission is a SKILL.md / AGENTS.md skill, please include in your PR description:

- **Compatibility** — Which tools does it work with? (Claude Code, Cursor, Windsurf, OpenClaw, etc.)
- **Scope** — What MegaETH-specific patterns does it cover?
- **Testing** — Has it been tested against MegaETH mainnet or testnet?

## Pull Request Process

1. Fork the repository
2. Create a branch for your addition
3. Make your changes to `README.md`
4. Submit a pull request with a clear description of what you're adding and why

## New Categories

If your item doesn't fit an existing category, you may suggest a new one. Explain why it's needed in your PR description.

## Questions?

Open an issue if you're unsure whether something belongs here.
